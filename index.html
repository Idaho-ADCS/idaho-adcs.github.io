Contents

This is the documentation of libraries and source code written for validating system design. It is not complete, as much of the code here represents code used to validate sensor and component function, actuator design, and dynamics of the system. More simply, this code proves our design works.

It implements enough functionality for the system to fly safely in space and allow the mother satellite to:

  1. Collect data and monitor system function including:
    • power consumption and system voltage
    • data collected by the sensors (Gyro, Magnetometer, Photodiodes)
  2. Test actuator function
    • drive the flywheel
    • drive magnetorquers
  3. Receive UART commands from satellite
    • move from STANDBY mode (default) to HEARTBEAT (periodic data collection) to actuator testing

Satellite Simulator

Hardware testing and validation requires a microgravity simulation where friction on the satellite is minimal so that its reaction to PID and other control systems can be carefully observed and documented. To facilitate this we developed a wireless test rig or "hanging by a thread" test. We used an ESP32 microcontroller to pose as the satellite and support the UART communication link with the ADCS system. It provided the enable signal, UART connection, and hosted a webserver which we used to tell the ESP32 when and what commands to send the ADCS as well as collect data from the ADCS over UART.
